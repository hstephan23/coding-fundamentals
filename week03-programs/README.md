# Week 03 - Control Flow

## Learning Goals 

- Programs make decisions 
- Programs repeat steps

### Mental Model 

Only one path runs at a time.

#### Prompts 

- Which path is taken and why? 
    
    The path taken depends on the value of i in that iteration.

    In this case, it takes the path based on being even or odd. When it is even it takes the if statement, when
    it is odd it takes the else statement. 

    It takes that path because it is the conditions sent to the computer.

- What condition controls this loop? 

    If i (essentially the counter) is less than or equal to 10, it will continue. Once it reaches 11, the for loop stops.
    
- How could I describe this logic in plain English?

    We want to find out if values between 1 and 10 are even or odd. We loop through the numbers from 1 to 10. And then 
    validate it based on the condition of if it is divisible by 2 with no remainder. This allows us to put in a condition
    that checks that. We can then print out the message depending on the result of that conditional check.
